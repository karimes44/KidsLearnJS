DrawBoard()

Just as for Tic-Tac-Toe we need a game board. But we need a BIG one this time, maybe with var Nc = 20 columns and var Nr = 10 rows, for a total of var Ne = Nc*Nr 200 cells.

To write an HTML table with so many cells, that would be annoying... Why don't we ask the computer to help us out?

We can do that by using JS loops to build up a looong string called TBL, that contains the HTML definition of our table with 200 cells:

 var TBL = "<table>";
 for (var ir=0;ir<Nr;ir++) {
  TBL += "<tr>";
  for (var ic=0;ic<Nc;ic++) {
   nid = ic + Nc*ir;
   TBL += "<td id='" + nid + "'>~</td>";
  }
  TBL += "</tr>";
 }
 TBL += "</table>";

We placed the "~" sign in each that, where the mole at the monent is not...

W3School: JS Loops For.

Play()

The function that lets you play the game, is quite simple and contains only things we saw already in earlier games:

We randomly pick a cell and write into it the mole, which is the "@" sign, as well as an onclick event that calls the Play(i), when you click on the mole.

We also have to clean-up after ourselves: We write into the cell where the mole was when we clicked it, the "~" to show that the mole is no longer there.

But at the beginning there is no mole? Right, that's why the Add Mole button calls Play(-1), that is with the cell index -1 that does not exists. So we have to check for that in the Play function...
