Flash Cards

Instructions

Overview

Flash cards are good for learning vocabulary:

For example, one side may show a word in English, and the other side shows the same word in Spanish. Then you can look and the English word and say out loud the Spanish word, and the turn the card over to see if you got it right.

How about an app which lets you make up your own flash cards?

  1. One side of the Flash Card is shown in blue-ish.
  2. Think of that is on the other side and click anywhere on the card.
  3. That shows the back side in green-ish, and you can see if you thought correctly.
  4. In the code, you'll see something like:
    var FlashCards = [['A', 'a'], ['B', 'b'], ['C', 'c'], ['D', 'd']]
  5. Replace the pairs like ['A', 'a'] with a vocabulary pair, such as ['the House', 'la Casa'].

<body>

Nothing new here, just a span placeholder to set an id, here FCard, so that the functions know where to place the generated HTML code.

<script>

We learned about an array that contained serval single characters, such as "a", "b", "c", in the Tic-Tac-Toe game earlier. Here we see and array that has arrays as elements. These arrays here have string values, one for the front of the flash card and one for the back. You will replace the dummy characters with the vocabulary words that you want to memorize.

NewCard()

The NewCard() function has a lot of similarities with the Tic-Tac-Toe code for one player: We check first if we have already looked at all the cards in our array. If not, we randomly pick one, and show it in the FCard placeholder.

We want to show only the front of the ith flash card. FlashCards[i] gets us to the right card, and FlashCards[i][0] is the front of this card.

We also add the function that is executed when you click the shown card: FlipCard(i) — we talk about that one later.

class='card front'

The card in shown inside a paragraph tag <p>, but we want to make this paragraph look special, like an index card. We again use CSS for that. To tell CSS which paragraph to display in the form of an index card, we give the paragraph a class attribute, here we add even two of them: "card" and "front" or "back".

In the CSS sections you see classes are defined by putting a dot "." in front of the name, so that we can keep them apart from regular tags. For the card class we set a lot of attributes: We want the text BIG and in the center. We want the card to show a boarder, with rounded corners, etc.

You can look again at W3School: CSS Tutorial to check what all these attributes do.

The class definitions for "front" or "back" are pretty simple: we just set a different color!

FlipCard()

The FlipCard() just shows the back side, i.e., it uses FlashCards[i][1], instead of FlashCards[i][0].

It also removes the flash card we have already looked at from the array using the splice method that we know from Tic-Tac-Toe already.
