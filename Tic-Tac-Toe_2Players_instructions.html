Tic-Tac-Toe for 2 Players

Instruction

<table>

For Tic-Tac-Toe we need a 3x3 game board. A good way to create that is by using an HTML Table.

You can find a good explanation of how tables are defined in HTML at W3School: HTML Tables.

<style>

... but we also want the cells of the table to be square! We use CSS for that:

<style>
...
td {
 margin:       5px;
 font-size:    60pt;
 width:        1em;
 text-align: center;
}
</style>
  1. The margin of 5 pixels makes sure that there is a bit of space between the cell border and the X or the O.
  2. We want a BIG fond, so we set the font-size to 60 points!
  3. To get a square cell we set the width of the cell to 1em, which means the size of an "M".
  4. We also want the X or O be in the center of the cell. That's what we do with the text-align option.

For more on CSS check out: W3School: CSS Tutorial

id and onclick

We need to make sure that our JavaScript program can find the right cells. For that we give each of the nine cells a different identifier or id. Here we just label them with the first 9 letters of the alphabet. Also, in JS you can click anything, so we define that when you click a cell the computer will call the function MarkCell(i), which takes the cell identifier as its argument:

<td id="c" onclick="MarkCell('c')">&nbsp;</td>

"c" or 'c'

Why did we use double quotes " in one place and single quote ' somewhere else?

Parameter values that are letters or words (remember? they are called strings) must be enclosed in quotes. You can use either, single or double, does not matter ... but if you need to put a string inside another string, then you must use the other type, so that the computer does not get confused about where they start and end.

To learns more about strings, check out: W3School: JS Strings

What is &nbsp;?

HTML ignores "while spaces", so in the beginning, when our table is still empty, the table would be collapsed - ugly!
... so we need to tell HTML not to ignore the space we put in. And &nbsp; does just that! nbsp stand for non-breaking space.

MarkCell(i)

The function MarkCell(i) uses only command you learned about already: It put first an "X" in the cell you clicked, and then toggles between "X" and "O" for the next time you click a cell.

Scope

A slightly confusing detail: We use var Mark = "X"; outside the function definition but drop the var inside the function definition. Why?

When you define a variable outside of a function or some other block, this variable can be used anywhere, it called a global variable. If you define the same variable inside a block with the var command, that you create a new variable that exists only in this block. That's called a local variable. Confusing at first, but you'll get the hang of it!

To learn more about global and local variable, check out: W3School: JS Scope
