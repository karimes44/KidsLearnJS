Now that you took a look at HTML and CSS, it's time to get to the last one of the three: JavaScript (or "JS" for short)

JavaScript lets you "interact" with the web page, that means you get to do something and the web page changes in response, that makes it more like a game or an app.

Khan Academy: What is Programming?

Now let's take a closer look at our first game Hello YOU. This game just greets you after you entered your name.
... but how does it actually do that?

  1. The in section <script> we define a function that is called SayHello(). This function does all the magic.
  2. The command:
    <textarea id="NAME" rows="1" cols="20"></textarea>
    creates the box in the game in which you can type your name. (The size of the text box is 1 "rows" high and 20 "columns" wide, that is space for 20 characters.)
  3. Once you are done typing your name you press the Submit button. This button is created with:
    <button type="button" onclick="SayHello()">Submit</button>
  4. onclick="SayHello()" means, that when you click the button, the function SayHello() will be called.
  5. The command:
    <span id='Dialog'></span>
    just marks the spot where the SayHello() function will display the response.
  6. The command:
    var NAME = document.getElementById("NAME").value;,
    allows the SayHello() function to read what you typed into the box and to store this text in a variable we called NAME.
  7. The command:
    TEXT = "<p>Hi <b>" + NAME + "</b>! It is nice to meet you.</p>";
    create a text variable (called TEXT) that contains the greeting as well as the variable with your name.
  8. The command: document.getElementById("Dialog").innerHTML = TEXT;
    writes the content of the TEXT variable into our web page in the spot that we marked in step 5.
  9. ... and this is all there is to it!

What are "strings" anyway?

In JS any kind of "text", when used in a variable, is calls a "string", because it is kind of a string of characters. All strings have to be enclosed in quotes, either double or single quotes. Let's look again the command:

NAME = "Alice",
TEXT = "<p>Hi <b>" + NAME + "</b>! It is nice to meet you.</p>";

NAME is a string variable that we set to "Alice". TEXT is also a string variable, but we build this string out of several parts:

  1. "<p>Hi <b>": A string that even contains HTML tags.
  2. NAME: A string variable, that will get replaced with its value, which here is "Alice".
  3. "</b>! It is nice to meet you.</p>": Another string that contains HTML tags.

... and we stick them together with "+" (for strings that is also called concatenating - just some fancy word for it...)

What is the value of the string variable TEXT the end?

<p>Hi <b>Alice</b>! It is nice to meet you.</p>

... and the browser will display that as:

Hi Alice! It is nice to meet you.
