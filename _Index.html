Learn Computer Programming with HTML, CSS and JavaScript

Introduction

You can learn how to program ("write code") through creating games and apps that run on any browser (like the one you use to watch You Tube videos, or write emails, or do anything you do on the Web - even on a smart phone). You can email them to your friends and they can play with them right away.

Together we'll look at how Web Pages are built and learn about HTML (that is how you get words on a web page) and CSS (that is how you make the words look good).

We'll also learn how you can interact with the web page. For example, you can click somewhere, and things happens or enter your name and the web page does something with it. JavaScript does that. With JavaScript the Web Page becomes a game or an app.

Technical Requirements

The games and apps you will write will run on any browser and you can use any text editor to write the games and apps.

You can use a PC, a Mac, or a Chrome Book (... even a Linux box) to write the games and apps.

On mobile devices like Smart Phones and Tablets you can run the games, but you can't really write them there. That's because it is hard to find good text editors for mobile devices and the mobile browser don't have Developer Tools.

Developer Tools

When you write code, it is very normal that it does not quite work the first time (...like never!), and frankly, figuring out what is wrong can be confusing (sigh) - but also a fun challenge!
So, don't get discouraged, there are simple things you can do to get your code to start working right.

For example, you can show the value of some variables that don't quite do what you expected in a pop-up alert box - in fact you'll see quite a few commented out (=not active) alert statements in the JavaScript code. You can un-comment (=make them active again) them as needed or add new once to show the variables you are confused about.

There are also the Developer Tools - most browsers have them built-in. In Chrome you can activate the Developer Tools with Ctrl-Shift-i, or from the customization tab > More Tools > Developer tools.

The Console panel in the Developer Tool window you can see a list of all coding mistakes that prevent the code from running at all.

Another Tip: Browsers are designed to "muddle through" if there is an error on some Web page. When your code behaves like "totally weird", it may just be that there is a mistake in your code somewhere else and the browser just tried to limp along. Check in the Developer Tools if you can find any hints of what may have tripped up the browser.

In the Source panel you can step through the running code and see all the variables with their current values and see in which order your code statements are executed. Then you learn pretty quick why things are not quite working as you wanted. And once you know what is going wrong - fixing it is often the easy part!
